# 04 - 自然语言处理 (NLP)

> 自然语言处理是人工智能皇冠上的明珠——让机器真正"读懂"人类的语言。

---

## 4.1 什么是自然语言处理

### 4.1.1 NLP 的定义

**自然语言处理（Natural Language Processing, NLP）** 是人工智能和语言学的交叉学科，核心目标是让计算机能够**理解、分析和生成**人类的自然语言。

我们日常使用的中文、英文、日语等都是"自然语言"，与编程语言（Python、Java）这类"人工语言"不同，自然语言充满了灵活性和模糊性。NLP 就是要在这种模糊性中找到确定性，让机器像人一样处理语言。

**生活中的 NLP 应用无处不在：**
- 搜索引擎理解你输入的查询意图
- 手机输入法的智能联想和纠错
- 智能客服自动回答你的问题
- 机器翻译（如 Google 翻译、DeepL）
- 语音助手（Siri、小爱同学）理解你的指令

### 4.1.2 NLP 的挑战

自然语言处理之所以困难，主要源于以下几个核心挑战：

**1. 歧义性（Ambiguity）**

同一个词或句子可能有多种含义。例如：
- "苹果很好吃" vs "苹果发布了新手机"——"苹果"指水果还是公司？
- "他背了一首诗"——"背"是"背诵"还是"背负"？
- "咬死猎人的狗"——是狗咬死了猎人，还是猎人的狗被咬死了？

**2. 上下文依赖**

语言的含义高度依赖上下文。"这个东西不错"在购物评论中是好评，但如果前文是"你觉得怎么样？"后文是"但我不会买"，含义就完全不同了。

**3. 语言多样性**

全球有数千种语言，每种语言有不同的语法结构、书写系统和表达习惯。中文没有空格分隔词语，阿拉伯语从右向左书写，日语混用多种文字系统——这些都给 NLP 带来了独特的挑战。

### 4.1.3 NLP 的主要任务分类

NLP 任务可以大致分为两类：
- **自然语言理解 (NLU)**：让机器"读懂"文本，如文本分类、情感分析、信息抽取
- **自然语言生成 (NLG)**：让机器"写出"文本，如机器翻译、文本摘要、对话生成

## 4.2 文本预处理

原始文本通常是"脏数据"，需要经过一系列预处理步骤才能被模型使用。这个过程就像做菜前的备料——洗菜、切菜、去皮。

### 4.2.1 分词 (Tokenization)

分词是将连续文本切分成一个个独立"词语"的过程。

**英文分词**相对简单，因为单词之间有空格：
```
"I love NLP" → ["I", "love", "NLP"]
```

**中文分词**则是一个重大挑战，因为中文没有天然的分隔符：
```
"南京市长江大桥" → "南京市 / 长江大桥" 还是 "南京 / 市长 / 江大桥"？
```

常用的中文分词工具包括：**jieba**（Python 最流行的中文分词库）、**HanLP**、**pkuseg** 等。

```python
import jieba
text = "自然语言处理是人工智能的重要方向"
words = jieba.lcut(text)
print(words)  # ['自然语言', '处理', '是', '人工智能', '的', '重要', '方向']
```

### 4.2.2 停用词去除

停用词是指那些出现频率极高但携带信息很少的词，如中文的"的、了、是、在"，英文的"the, is, a, an"。去除停用词可以减少噪音，提高后续处理效率。

```
原文：["自然语言", "处理", "是", "人工智能", "的", "重要", "方向"]
去除停用词后：["自然语言", "处理", "人工智能", "重要", "方向"]
```

### 4.2.3 词干提取与词形还原

这两个概念主要针对英文等屈折语言：
- **词干提取 (Stemming)**：粗暴地去掉词缀，如 "running" → "run"，"happily" → "happili"（可能产生非标准词）
- **词形还原 (Lemmatization)**：还原为字典中的标准形式，如 "better" → "good"，"went" → "go"（更准确但更慢）

中文没有词形变化，因此一般不需要这两个步骤。

### 4.2.4 文本清洗

实际文本中通常混杂着各种"杂质"，需要清洗处理：
- 去除 HTML 标签、特殊字符、多余空格
- 统一大小写（英文）
- 去除或替换数字、URL、邮箱地址
- 繁简转换（中文）
- 处理表情符号和特殊 Unicode 字符

## 4.3 文本表示方法

计算机无法直接理解文字，我们必须将文本转换为**数字向量**，这个过程叫做文本表示（Text Representation）。文本表示方法的发展历程，本身就是一部 NLP 的进化史。

### 4.3.1 独热编码 (One-Hot Encoding)

最简单的表示方法：每个词用一个很长的向量表示，向量中只有一个位置是 1，其余全是 0。

假设词汇表为 ["猫", "狗", "鱼"]：
```
猫 → [1, 0, 0]
狗 → [0, 1, 0]
鱼 → [0, 0, 1]
```

**缺点**：向量极度稀疏（如果词汇表有 10 万个词，每个向量就有 10 万维）；无法表示词与词之间的语义关系（"猫"和"狗"的距离 = "猫"和"经济"的距离）。

### 4.3.2 词袋模型 (Bag of Words, BoW)

将文本看作一个"装词的袋子"，只关心每个词出现的次数，不关心词的顺序。

```
文档1："我 喜欢 猫 我 喜欢 狗" → {我:2, 喜欢:2, 猫:1, 狗:1}
文档2："他 不 喜欢 猫"          → {他:1, 不:1, 喜欢:1, 猫:1}
```

**缺点**：丢失了词序信息——"猫追狗"和"狗追猫"的表示完全一样。

### 4.3.3 TF-IDF

**TF-IDF（词频-逆文档频率）** 是对词袋模型的改进，核心思想是：一个词的重要性不仅取决于它在当前文档中出现的频率（TF），还取决于它在整个语料库中的稀有程度（IDF）。

- **TF（词频）**= 词在文档中出现的次数 / 文档总词数
- **IDF（逆文档频率）**= log(总文档数 / 包含该词的文档数)
- **TF-IDF = TF × IDF**

举例："机器学习"在一篇 AI 论文中出现多次（TF 高），且不是所有文档都包含它（IDF 较高），所以 TF-IDF 值高，说明它对这篇文档很重要。而"的"虽然 TF 高，但几乎所有文档都有它（IDF 低），所以 TF-IDF 值低。

### 4.3.4 Word2Vec

2013 年，Google 提出的 **Word2Vec** 是词向量领域的革命性工作。它将每个词映射到一个**低维稠密向量**（如 100-300 维），语义相近的词在向量空间中距离相近。

Word2Vec 有两种训练架构：
- **CBOW（连续词袋）**：根据上下文词预测中心词。如给定"今天 天气 __ 好"，预测"很"。训练速度快，适合频繁出现的词。
- **Skip-Gram**：根据中心词预测上下文词。如给定"很"，预测"今天""天气""好"。对低频词效果更好。

Word2Vec 最令人惊叹的特性是**语义类比**：
```
向量("国王") - 向量("男人") + 向量("女人") ≈ 向量("王后")
```

### 4.3.5 GloVe

**GloVe（Global Vectors）** 由斯坦福大学于 2014 年提出。与 Word2Vec 使用局部上下文窗口不同，GloVe 利用**全局词共现矩阵**来训练词向量。它结合了全局统计信息和局部上下文的优势，在很多任务上表现优异。

### 4.3.6 从静态词向量到动态上下文表示

Word2Vec 和 GloVe 都是**静态词向量**——每个词只有一个固定的向量表示，无论出现在什么上下文中。这个缺陷显而易见："bank"在"river bank"（河岸）和"bank account"（银行账户）中含义不同，但向量却相同。

后来的 **ELMo、BERT、GPT** 等模型解决了这个问题，它们生成的词向量是**上下文相关的**——同一个词在不同句子中有不同的表示。这是 NLP 发展的重要里程碑。

## 4.4 NLP 经典任务

### 4.4.1 文本分类

将文本自动归入预定义的类别。这是最基础也最广泛的 NLP 任务。

典型应用：
- **情感分析**：判断评论是正面还是负面。"这家餐厅太好吃了！" → 正面；"服务态度极差" → 负面
- **垃圾邮件检测**：判断邮件是正常邮件还是垃圾邮件
- **新闻分类**：将新闻自动归入体育、科技、娱乐等类别

### 4.4.2 命名实体识别 (NER)

从文本中识别出**人名、地名、机构名、时间**等实体。

```
"乔布斯在2007年于旧金山发布了iPhone"
→ 人名：乔布斯 | 时间：2007年 | 地名：旧金山 | 产品名：iPhone
```

NER 是信息抽取的基础，广泛应用于知识图谱构建、搜索引擎和智能问答。

### 4.4.3 关系抽取

在识别实体的基础上，进一步判断实体之间的关系。

```
"乔布斯是苹果公司的创始人"
→ (乔布斯, 创始人, 苹果公司)
```

### 4.4.4 机器翻译

将一种语言的文本自动翻译为另一种语言。从早期的基于规则的方法，到统计机器翻译，再到如今的**神经机器翻译（NMT）**，机器翻译的质量已经有了质的飞跃。Google 翻译、DeepL 等都是典型代表。

现代机器翻译主要基于 **Encoder-Decoder + Attention** 架构或 **Transformer** 架构。

### 4.4.5 文本摘要

将长文本自动压缩为短摘要，分为两种方法：
- **抽取式摘要**：从原文中直接挑选关键句子组成摘要
- **生成式摘要**：理解原文后，用新的语句重新表述核心内容（更接近人类的做法，但难度更大）

### 4.4.6 问答系统

根据用户的自然语言问题，自动给出答案。分为：
- **抽取式问答**：从给定文本中找到包含答案的片段（如 SQuAD 任务）
- **生成式问答**：根据知识生成完整的答案（如 ChatGPT）
- **知识库问答（KBQA）**：将问题转化为对知识图谱的查询

### 4.4.7 对话系统

与人类进行自然语言交互的系统，分为：
- **任务导向型对话**：完成特定任务（如订机票、查天气）
- **开放域对话**：自由闲聊，不限定话题（如 ChatGPT、文心一言）

## 4.5 预训练语言模型

预训练语言模型是近年来 NLP 最重大的突破，它彻底改变了 NLP 研究和应用的范式。

### 4.5.1 ELMo：上下文词向量的开端

**ELMo（2018）** 是第一个广泛使用的上下文词向量模型。它使用双向 LSTM 语言模型，让同一个词在不同上下文中获得不同的向量表示。

```
"他在河边的bank钓鱼"  →  bank 的向量偏向"河岸"
"他去bank取钱"        →  bank 的向量偏向"银行"
```

ELMo 首次证明了：**先在大规模语料上预训练，再在特定任务上微调**的范式是有效的。

### 4.5.2 BERT：双向编码器

**BERT（2018，Google）** 是 NLP 领域的里程碑。它基于 **Transformer 的编码器**架构，通过两个巧妙的预训练任务来学习语言表示：

- **掩码语言模型 (MLM)**：随机遮挡句子中 15% 的词，让模型预测被遮挡的词。如"我喜欢吃[MASK]果" → "苹"
- **下一句预测 (NSP)**：判断两个句子是否连续

BERT 的关键创新是**双向**——它同时利用左右两侧的上下文来理解每个词。BERT 在发布时刷新了 11 项 NLP 任务的最佳记录，引发了预训练模型的研究热潮。

### 4.5.3 GPT 系列：自回归生成模型

**GPT（Generative Pre-trained Transformer，OpenAI）** 采用 **Transformer 的解码器**架构，使用自回归方式训练——根据前面的词预测下一个词。

- **GPT-1（2018）**：证明了生成式预训练的有效性
- **GPT-2（2019）**：扩大模型规模，展示了惊人的文本生成能力
- **GPT-3（2020）**：拥有 1750 亿参数，展示了强大的少样本学习能力（few-shot learning）
- **GPT-4（2023）**：多模态模型，能力进一步大幅提升

GPT 系列证明了一个重要规律：**模型规模的扩大会带来能力的质变**（Scale Law）。

### 4.5.4 T5：统一文本到文本框架

**T5（Text-to-Text Transfer Transformer，Google，2020）** 提出了一个优雅的统一框架：将所有 NLP 任务都转换为"文本到文本"的格式。

```
翻译：     "translate English to French: Hello" → "Bonjour"
摘要：     "summarize: 一段很长的文章..." → "摘要内容"
分类：     "classify: 这部电影很精彩" → "正面"
问答：     "question: 中国的首都? context: 北京是中国的首都" → "北京"
```

这种统一框架的优势是：一个模型可以处理所有任务，极大简化了系统设计。

### 4.5.5 预训练 + 微调范式的意义

传统做法是为每个任务从零开始训练模型，既费时又需要大量标注数据。**预训练 + 微调**范式改变了这一切：

1. **预训练阶段**：在海量无标注文本上学习通用语言知识（成本高，但只需做一次）
2. **微调阶段**：在少量标注数据上针对特定任务进行调整（成本低，效果好）

这就像人类先在学校学习通用知识（预训练），然后到工作岗位上快速掌握专业技能（微调）。

## 4.6 现代 NLP 技术趋势

随着大语言模型的兴起，NLP 正在经历新一轮的范式变革。

### 4.6.1 Prompt Engineering（提示工程）

传统微调需要修改模型参数。**Prompt Engineering** 则不修改模型，而是通过设计精巧的提示（Prompt）来引导模型输出期望的结果。

```
普通提问："翻译这句话"
优化 Prompt："你是一位资深的中英翻译专家。请将以下中文翻译为地道的英文，
             保持原文的语气和风格：..."
```

好的 Prompt 可以显著提升模型表现。Prompt Engineering 已经成为使用大语言模型的核心技能。

### 4.6.2 In-Context Learning（上下文学习）

GPT-3 展示了一种神奇能力：不需要微调，只需在输入中给出几个示例，模型就能"学会"新任务。

```
将以下英文翻译为中文：
英文：Hello → 中文：你好
英文：Thank you → 中文：谢谢
英文：Good morning → 中文：
```

模型会自动理解模式并输出"早上好"。这种能力叫做 **few-shot learning（少样本学习）**，它使得无需训练就能适应新任务成为可能。

### 4.6.3 指令微调 (Instruction Tuning)

在大量不同任务的"指令-回复"数据对上微调模型，使模型更好地理解和遵循人类指令。

```
指令："请用三句话总结以下文章的核心观点"
指令："请以5岁小孩能理解的方式解释量子力学"
指令："请找出以下代码中的 bug 并修复"
```

经过指令微调的模型（如 ChatGPT、文心一言）能够理解各种格式的指令，通用性大大增强。

### 4.6.4 RLHF（人类反馈强化学习）

**RLHF（Reinforcement Learning from Human Feedback）** 是让 ChatGPT 表现如此出色的关键技术之一。其核心流程：

1. **监督微调（SFT）**：用人工标注的高质量对话数据微调模型
2. **训练奖励模型（RM）**：让人类对模型的多个回复进行排序，训练一个"评分模型"
3. **强化学习优化（PPO）**：使用奖励模型的反馈信号，通过强化学习进一步优化语言模型

简单来说，RLHF 就是让人类"教"模型什么是好的回复、什么是不好的回复，从而使模型的输出更符合人类偏好——更准确、更安全、更有帮助。

1. **入门阶段**：掌握 Python 基础 → 学习文本预处理和基本文本表示 → 用 scikit-learn 完成简单的文本分类任务
