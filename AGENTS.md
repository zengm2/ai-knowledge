# AI Knowledge 项目规则

## QA 自动更新规则

每次用户提出与 AI 学习相关的问题并得到解答后，**必须自动执行以下操作**：

1. **创建 QA 文档**：在 `QA-常见问题/` 目录下，以问题关键词作为文件名（如 `xxx关键词.md`），写入完整的问答内容
2. **QA 文档格式要求**：
   - 标题：`# Q: 原始问题`
   - 关键词标签（blockquote）
   - 简短回答（一句话版本）
   - 详细解答（完整分析，含表格、代码、图示等）
   - 相关章节链接
3. **更新总大纲**：在 `README.md` 的 "QA 常见问题" 表格中追加新条目（编号、链接、关键词）
4. 以上操作在回答完用户问题后**自动完成**，无需用户提醒

## 文件命名规则（强制）

为保证 GitHub 上 Markdown 链接正常工作，**所有文件名必须遵守以下规则**：

1. **禁止使用空格**：文件名中不得包含空格字符
   - 英文单词之间用连字符 `-` 连接，如 `AI-Agent与RAG的关系.md`
   - 或直接拼接，如 `ScalingLaws.md`
2. **中文与英文之间**：直接拼接，不加空格，如 `Devin-AI软件工程师分析.md`
3. **禁止使用的字符**：空格、`#`、`%`、`&`、`{`、`}`、`\`、`<`、`>`、`*`、`?`、`$`、`!`、`'`、`"`、`@`、`+`、`` ` ``、`|`、`=`
4. **允许使用的字符**：中文、英文字母、数字、连字符 `-`、下划线 `_`、点 `.`

**命名示例**：

```
✅ 正确：AI-Agent与RAG的关系.md
✅ 正确：Manus-Agent分析.md
✅ 正确：幂律关系与ScalingLaws.md
✅ 正确：检索增强生成RAG.md

❌ 错误：AI Agent与RAG的关系.md     （含空格）
❌ 错误：Manus Agent分析.md          （含空格）
❌ 错误：幂律关系与Scaling Laws.md   （含空格）
```

## 链接检查

项目根目录下提供了 `check-links.sh` 脚本，用于验证 README.md 中所有链接是否指向存在的文件。

**使用方法**：

```bash
cd ai-knowledge/
bash check-links.sh
```

**建议在以下时机运行**：
- 新增 QA 文档后
- 重命名文件后
- 提交到 Git 之前
