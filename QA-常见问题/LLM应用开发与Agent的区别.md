# Q: LLM 应用开发和 Agent 有什么区别？

> **关键词**：LLM 应用开发、AI Agent、工具调用、自主决策、行动循环、规划、RAG、链式调用、复杂度光谱

**简短回答**：LLM 应用开发是大范畴，包含所有用大模型构建产品的工作；Agent 是其中最复杂的一种形态，核心区别在于 Agent 能自主规划、选择工具、循环行动、从错误中恢复。普通 LLM 应用的流程是人写死的，Agent 的流程是 AI 自己决定的。

---

## 详细解答

### 关系

```
LLM 应用开发 = 用大模型构建各种产品（大范畴）
Agent       = LLM 应用的一种特定形态（能自主决策和行动）
```

Agent 是 LLM 应用的子集，不是所有 LLM 应用都是 Agent。

### 核心区别

| 特征 | 普通 LLM 应用 | Agent |
|------|-------------|-------|
| **流程控制** | 人预设的固定流程 | AI 自己决定下一步 |
| **自主规划** | 无 | 有 |
| **工具使用** | 无或有限 | 多种工具，自主选择 |
| **循环决策** | 单次调用 | 多次循环 |
| **环境交互** | 通常只读 | 能改变外部世界 |
| **错误恢复** | 返回错误给用户 | 自动重试和调整策略 |

### 技术公式

```
普通 LLM 应用 = LLM + Prompt + (可选 RAG)
Agent         = LLM + 规划 + 工具 + 记忆 + 行动循环
```

### 复杂度光谱

| 层级 | 特点 | 举例 |
|------|------|------|
| **L1: 单次调用** | 一进一出 | 翻译 API、文本分类 |
| **L2: 链式调用** | 多步固定流程 | 先提取关键词再搜索再总结 |
| **L3: RAG 应用** | 检索 + 生成 | 企业知识库问答 |
| **L4: 简单 Agent** | 有限自主决策 | ChatGPT 联网搜索 |
| **L5: 复杂 Agent** | 完整规划-执行-验证循环 | Devin、Manus、OpenClaw |

### 技术栈对比

| 维度 | 普通 LLM 应用 | Agent |
|------|-------------|-------|
| **框架** | LangChain、LlamaIndex | LangGraph、CrewAI、AutoGen |
| **开发难度** | 相对简单 | 较高 |
| **成本** | 低 | 高（多次循环调用） |

### 学习路径

```
先学 LLM 应用开发 → Prompt → API → RAG → 评估优化
再进阶 Agent 开发 → 工具调用 → 规划决策 → 记忆 → 多 Agent
```

### 一句话总结

大多数实际项目需要的是 L1-L3 的 LLM 应用能力，Agent 是前沿但还在成熟中。
