# Q: 向量数据库是什么，怎么组成的，怎么工作的？

> **关键词**：向量数据库、Vector Database、Embedding、相似度搜索、ANN、HNSW、余弦相似度、FAISS、Chroma

**简短回答**：向量数据库是专门存储和搜索"向量"的数据库，能根据语义相似度快速找到最相关的内容。它由存储层（向量+元数据）、索引层（ANN 算法加速）、查询引擎（相似度计算）组成，是 RAG、语义搜索等 AI 应用的基础设施。

---

## 详细解答

### 什么是"向量"

向量就是一组数字，用来表示一段文本的"语义含义"：

```
"今天天气很好"  →  [0.12, -0.34, 0.87, ...]   （如 1536 维）
"天气不错"      →  [0.11, -0.32, 0.85, ...]   （非常接近）
"股票大跌"      →  [-0.78, 0.45, -0.12, ...]   （差距很大）
```

语义相近的文本，向量在空间中的距离也近。

### 向量数据库的组成

```
┌────────────────────────────────────────┐
│            向量数据库                    │
│                                          │
│  1. 存储层 — 向量数据 + 元数据 + ID       │
│  2. 索引层 — ANN 索引结构（HNSW 等）      │
│  3. 查询引擎 — 相似度计算 + Top-K + 过滤  │
│  4. API 层 — 插入/查询/删除/更新          │
└────────────────────────────────────────┘
```

每条数据的结构：

```json
{
  "id": "doc_001",
  "vector": [0.12, -0.34, 0.87, ...],
  "metadata": {
    "text": "员工入职满1年，享有5天带薪年假",
    "source": "员工手册.pdf",
    "page": 15
  }
}
```

### 工作流程

**写入**：原始文本 → Embedding 模型转向量 → 存入数据库 → 建立索引

**查询**：查询文本 → 同一模型转向量 → 在数据库中搜索最相似向量（Top-K）→ 返回结果

### 相似度计算

| 方法 | 说明 | 特点 |
|------|------|------|
| **余弦相似度** | 两向量夹角的余弦值 | 最常用，1 = 完全相同 |
| **欧氏距离** | 两点直线距离 | 值越小越相似 |
| **点积** | 对应元素乘积之和 | 适合归一化向量 |

### 索引算法（ANN 近似最近邻）

| 算法 | 原理 | 特点 |
|------|------|------|
| **HNSW** | 多层图结构跳跃搜索 | 最常用，查询快 |
| **IVF** | 先聚类分桶再搜索 | 适合大规模 |
| **PQ** | 向量压缩成短编码 | 省内存 |

### 向量数据库 vs 传统数据库

| 对比 | 传统数据库 | 向量数据库 |
|------|-----------|-----------|
| **存什么** | 结构化数据 | 高维向量 + 元数据 |
| **怎么查** | SQL 精确匹配 | 相似度搜索 |
| **索引** | B-Tree、Hash | HNSW、IVF |
| **场景** | 业务数据管理 | AI 语义搜索 |

### 主流产品

| 产品 | 类型 | 适合场景 |
|------|------|---------|
| **Chroma** | 开源 | 学习/原型验证 |
| **pgvector** | PG 扩展 | 已有 PostgreSQL |
| **Qdrant** | 开源 | 生产环境中小规模 |
| **Milvus** | 开源 | 生产环境大规模 |
| **Pinecone** | 云服务 | 免运维 |
| **FAISS** | 搜索库（Meta） | 算法研究 |

### 代码示例（Chroma）

```python
import chromadb

client = chromadb.Client()
collection = client.create_collection("docs")

# 写入
collection.add(
    documents=["员工入职满1年，享有5天带薪年假", "年假需提前3个工作日申请"],
    ids=["doc1", "doc2"]
)

# 查询
results = collection.query(query_texts=["年假有几天"], n_results=2)
print(results["documents"])
```

### 一句话总结

向量数据库把文本转成数字向量存储，查询时通过计算向量间距离找到语义最相似的内容，是 RAG 和语义搜索的核心基础设施。
