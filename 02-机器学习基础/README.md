# 02 - 机器学习基础

> 本章系统介绍机器学习的核心概念、三大范式、模型评估方法、特征工程以及经典算法，适合零基础入门学习者。

## 2.1 什么是机器学习

### 定义

**机器学习（Machine Learning, ML）** 是人工智能的一个分支，它让计算机能够**从数据中自动学习规律和模式**，而不需要被显式地编程。简单来说，机器学习就是"用数据训练出能做预测或决策的模型"。

> 经典定义（Tom Mitchell, 1997）：如果一个程序在任务 T 上的性能 P 随着经验 E 的增加而提高，那么我们就说这个程序在从经验中学习。

### 机器学习与传统编程的区别

| | 传统编程 | 机器学习 |
|---|---|---|
| **输入** | 规则 + 数据 | 数据 + 期望结果 |
| **输出** | 结果 | 规则（模型） |
| **举例** | 程序员手写垃圾邮件过滤规则 | 算法从大量邮件样本中自动学会识别垃圾邮件 |

**形象比喻**：传统编程就像老师给学生一份标准答案让他照着做；机器学习则像给学生大量练习题和参考答案，让他自己总结出解题方法。

### 机器学习的基本流程

```
数据收集 → 数据预处理 → 特征工程 → 模型选择与训练 → 模型评估 → 模型部署
```

1. **数据收集**：获取高质量、足够量的训练数据
2. **数据预处理**：清洗缺失值、处理异常值、格式转换
3. **特征工程**：从原始数据中提取对预测有用的特征
4. **模型训练**：选择合适的算法，用训练数据拟合模型
5. **模型评估**：用测试数据验证模型效果
6. **模型部署**：将模型上线，应用于实际场景

## 2.2 机器学习的三大范式

### 2.2.1 监督学习 (Supervised Learning)

**核心思想**：从"带标签"的数据中学习。训练数据包含输入（特征）和对应的输出（标签），模型学习从输入到输出的映射关系。

> 类比：就像有老师批改作业——每道题都有标准答案，学生通过对答案来学习。

**两大任务类型**：

| 类型 | 目标 | 输出 | 举例 |
|---|---|---|---|
| **分类（Classification）** | 预测离散类别 | 类别标签 | 邮件是否为垃圾邮件、图片中是猫还是狗 |
| **回归（Regression）** | 预测连续数值 | 实数值 | 预测房价、预测股票走势 |

**经典算法一览**：

- **线性回归（Linear Regression）**：用一条直线（或超平面）拟合数据，找到特征与目标值之间的线性关系
- **逻辑回归（Logistic Regression）**：虽然名字带"回归"，但实际上是分类算法，通过 Sigmoid 函数将线性结果映射到 0-1 概率
- **决策树（Decision Tree）**：像一棵倒立的树，通过一系列"是/否"问题逐步做出决策
- **随机森林（Random Forest）**：多棵决策树的"投票"集成，少数服从多数，显著减少过拟合
- **支持向量机（SVM）**：寻找能最大化两类数据间隔的最优分界面
- **K近邻（KNN）**：找到与新样本最近的 K 个邻居，以多数邻居的类别作为预测结果

### 2.2.2 无监督学习 (Unsupervised Learning)

**核心思想**：从"无标签"的数据中发现隐藏的结构和规律。数据没有标准答案，算法需要自己探索数据中的模式。

> 类比：就像没有老师的自学——面对一堆资料，学生自己归纳总结出知识体系。

**主要任务类型**：

**1. 聚类（Clustering）**：将相似的数据分到同一组

- **K-Means**：预先指定 K 个簇，通过反复迭代将每个数据点分配到最近的簇中心，然后更新簇中心，直到收敛。例如：将客户按消费习惯分成几类人群。
- **层次聚类（Hierarchical Clustering）**：自底向上不断合并最相似的簇（凝聚法），或自顶向下不断分裂（分裂法），形成树状结构（树状图）。

**2. 降维（Dimensionality Reduction）**：在保留关键信息的前提下减少特征数量

- **PCA（主成分分析）**：找到数据方差最大的方向，将高维数据投影到低维空间。例如：将 100 维的特征压缩到 10 维，同时保留大部分信息。
- **t-SNE**：一种非线性降维方法，特别擅长在 2D/3D 空间中可视化高维数据的聚类结构。

**3. 关联规则（Association Rules）**：发现数据中频繁共现的模式

- 经典案例："买了啤酒的顾客往往也会买尿布"——这就是从超市交易数据中挖掘出的关联规则（Apriori 算法）。

### 2.2.3 强化学习 (Reinforcement Learning)

**定义**：智能体（Agent）在环境（Environment）中通过不断**试错**（trial and error），学习如何采取行动以**最大化累积奖励**。

**核心概念**：

| 概念 | 说明 | 举例（下棋AI） |
|---|---|---|
| **状态（State）** | 环境的当前状况 | 棋盘上棋子的布局 |
| **动作（Action）** | 智能体可以采取的操作 | 在某个位置落子 |
| **奖励（Reward）** | 环境对动作的反馈信号 | 赢棋 +1，输棋 -1 |
| **策略（Policy）** | 从状态到动作的映射规则 | 在当前局面下应该走哪一步 |

**典型应用场景**：
- **游戏 AI**：AlphaGo 击败围棋世界冠军、OpenAI Five 打 Dota2
- **机器人控制**：机械臂抓取物体、自主行走
- **自动驾驶**：学习安全高效的驾驶策略
- **推荐系统**：根据用户反馈动态调整推荐策略
- **大模型对齐**：RLHF（基于人类反馈的强化学习）是训练 ChatGPT 等大模型的关键技术
- **多智能体协作**：多个 AI 智能体在共享环境中学习合作与竞争策略

## 2.3 模型评估与选择

### 数据集划分

将数据划分为三部分是机器学习的基本实践：

| 数据集 | 占比（参考） | 用途 |
|---|---|---|
| **训练集（Training Set）** | 60%-80% | 用于模型学习参数 |
| **验证集（Validation Set）** | 10%-20% | 用于调整超参数、选择模型 |
| **测试集（Test Set）** | 10%-20% | 用于最终评估模型的泛化能力 |

> **关键原则**：测试集在模型训练过程中绝对不能被使用，否则评估结果不可信。

### 过拟合与欠拟合

- **过拟合（Overfitting）**：模型在训练数据上表现极好，但在新数据上表现差。就像一个学生把所有练习题的答案都背下来了，但碰到新题就不会做。
  - **解决方法**：增加数据量、正则化（L1/L2）、Dropout、早停（Early Stopping）、减少模型复杂度
- **欠拟合（Underfitting）**：模型过于简单，在训练数据和新数据上表现都差。就像学生完全没学会知识点，练习题和考试题都做不好。
  - **解决方法**：增加特征、使用更复杂的模型、减少正则化强度

### 偏差-方差权衡（Bias-Variance Tradeoff）

- **偏差（Bias）**：模型预测值与真实值的差距。高偏差意味着模型过于简单（欠拟合）。
- **方差（Variance）**：模型对不同训练数据的敏感度。高方差意味着模型过于复杂（过拟合）。
- **权衡**：理想模型需要在偏差和方差之间找到平衡点——模型足够复杂以捕捉数据规律，又不至于复杂到记住噪声。

### 交叉验证（Cross-Validation）

**K 折交叉验证**是最常用的方法：将数据分成 K 份，每次用其中 1 份做验证、其余 K-1 份做训练，重复 K 次，取平均结果。

```
第1轮: [验证] [训练] [训练] [训练] [训练]
第2轮: [训练] [验证] [训练] [训练] [训练]
第3轮: [训练] [训练] [验证] [训练] [训练]
第4轮: [训练] [训练] [训练] [验证] [训练]
第5轮: [训练] [训练] [训练] [训练] [验证]
```

优点：充分利用有限数据，评估结果更稳定可靠。

### 常用评估指标

**分类任务指标**：

| 指标 | 公式 / 含义 | 适用场景 |
|---|---|---|
| **准确率（Accuracy）** | 正确预测数 / 总数 | 类别均衡时 |
| **精确率（Precision）** | TP / (TP + FP) | 关注"预测为正的有多少是对的"，如垃圾邮件过滤 |
| **召回率（Recall）** | TP / (TP + FN) | 关注"实际为正的有多少被找到"，如疾病检测 |
| **F1 分数** | 2 × Precision × Recall / (Precision + Recall) | 精确率和召回率的调和平均 |
| **AUC-ROC** | ROC 曲线下面积 | 衡量模型在不同阈值下的综合分类能力 |

**回归任务指标**：

| 指标 | 含义 |
|---|---|
| **MSE（均方误差）** | 预测值与真实值差的平方的均值，对大误差惩罚更重 |
| **MAE（平均绝对误差）** | 预测值与真实值差的绝对值的均值 |
| **R²（决定系数）** | 模型解释了多大比例的方差，越接近 1 越好 |

## 2.4 特征工程

> "数据和特征决定了机器学习的上限，而模型和算法只是逼近这个上限。" ——业界共识

> **注**：这一观点主要针对传统机器学习。在深度学习和大模型时代，模型能够自动学习特征表示（表示学习），手动特征工程的重要性有所降低。但在数据量有限、计算资源受限的场景中，特征工程仍然至关重要。

### 为什么特征工程重要？

特征工程是将原始数据转化为模型可以有效利用的特征的过程。好的特征能让简单模型也取得优异效果，而差的特征即使用再复杂的模型也难以补救。例如，预测房价时，"房屋面积"是一个好特征，而"门牌号"通常不是。

### 特征选择

从已有特征中挑选出最有用的子集，去除冗余或无关特征：

- **过滤法（Filter）**：通过统计指标（如相关系数、卡方检验）独立评估每个特征的重要性
- **包裹法（Wrapper）**：用模型性能作为标准，尝试不同的特征组合（如递归特征消除 RFE）
- **嵌入法（Embedded）**：在模型训练过程中自动选择特征（如 Lasso 回归的 L1 正则化、决策树的特征重要性）

### 特征提取

从原始数据中构造新的特征：

- **文本数据**：词频（TF-IDF）、词袋模型、Word2Vec 词向量
- **图像数据**：像素值、边缘检测、卷积特征
- **时间序列**：滑动窗口统计量、趋势特征、周期特征
- **现代方法**：预训练语言模型（BERT、GPT）生成的上下文相关词向量，以及 Sentence Transformers 生成的句子级嵌入表示，已大幅超越传统特征提取方法

### 数据预处理

| 方法 | 说明 | 适用场景 |
|---|---|---|
| **归一化（Min-Max）** | 将数据缩放到 [0, 1] 区间 | 对数据范围敏感的算法（如 KNN、神经网络） |
| **标准化（Z-Score）** | 将数据转换为均值为 0、标准差为 1 | 大多数算法，尤其是基于距离的算法 |
| **缺失值处理** | 删除、均值/中位数填充、模型预测填充 | 数据不完整时 |
| **异常值处理** | 基于统计方法（如 3σ 原则）或箱线图检测并处理 | 数据包含极端值时 |
| **类别编码** | 独热编码（One-Hot）、标签编码（Label Encoding） | 处理分类型特征 |

## 2.5 经典算法详解

### 线性回归（Linear Regression）

**原理**：假设特征和目标值之间存在线性关系，通过最小化预测值与真实值之间的均方误差（MSE）来找到最优参数。数学表达：`y = w₁x₁ + w₂x₂ + ... + b`。

**适用场景**：房价预测、销量预测等连续数值的预测任务；特征与目标之间近似线性关系时效果好。

**优点**：简单直观、可解释性强、训练速度快。  
**缺点**：只能捕捉线性关系，对非线性数据拟合能力弱。

> 例子：已知房屋面积和卧室数量，预测房屋售价。

### 逻辑回归（Logistic Regression）

**原理**：在线性回归的基础上加了一个 Sigmoid 函数 `σ(z) = 1/(1+e⁻ᶻ)`，将输出映射到 (0, 1) 区间表示概率，从而用于二分类任务。

**适用场景**：二分类问题（也可扩展为多分类），如邮件分类、疾病诊断、用户流失预测。

**优点**：简单高效、输出有概率含义、可解释性好。  
**缺点**：假设特征与对数几率之间是线性关系，处理复杂非线性关系能力有限。

> 例子：根据患者体检数据，预测是否患有某种疾病（是/否）。

### 决策树与随机森林

**决策树（Decision Tree）**：

**原理**：通过递归地选择最优特征进行分裂，构建一棵树状结构。每个内部节点是一个特征判断条件（如"年龄 > 30？"），叶子节点是最终预测结果。常用分裂准则：信息增益（ID3）、基尼系数（CART）。

**优点**：直观易懂、无需特征缩放、能处理非线性关系。  
**缺点**：容易过拟合，对数据的小变化敏感。

**随机森林（Random Forest）**：

**原理**：构建多棵决策树（每棵树用随机采样的数据和随机选择的特征子集训练），最终结果取所有树的"投票"结果（分类）或平均值（回归）。

**优点**：大幅减少过拟合、准确率高、能评估特征重要性。  
**缺点**：模型较大、训练时间较长、可解释性不如单棵决策树。

> 例子：银行根据客户的年龄、收入、信用记录等特征，决定是否批准贷款申请。

### 支持向量机（SVM）

**原理**：在特征空间中寻找一个最优超平面，使得两类数据之间的**间隔（Margin）最大化**。对于线性不可分的数据，通过**核函数（Kernel Trick）** 将数据映射到高维空间，使其变得线性可分。

**适用场景**：中小规模数据集的分类任务，特别是高维数据（如文本分类）。

**优点**：在高维空间中表现优秀、泛化能力强、核函数机制灵活。  
**缺点**：大规模数据上训练慢、对参数和核函数选择敏感、不直接输出概率。

> 例子：手写数字识别——将 28×28 像素的图片分类为 0-9 中的某个数字。

### K近邻（KNN）

**原理**：最直觉的算法之一——对于一个新样本，找到训练集中离它最近的 K 个样本（"邻居"），以这些邻居中出现最多的类别作为预测结果（分类），或取邻居目标值的平均值（回归）。

**适用场景**：小规模数据集、数据分布不规则的分类和回归任务。

**优点**：简单直观、无需训练过程、天然支持多分类。  
**缺点**：预测时计算量大（需要计算与所有训练样本的距离）、对特征缩放敏感、高维数据表现下降（维度灾难）。

> 例子：根据一部电影的类型特征（动作场景数、浪漫场景数），判断它属于动作片还是爱情片。

### 朴素贝叶斯（Naive Bayes）

**原理**：基于贝叶斯定理 `P(A|B) = P(B|A) × P(A) / P(B)`，在"特征之间相互独立"的朴素假设下，计算样本属于每个类别的后验概率，选择概率最大的类别作为预测结果。

**适用场景**：文本分类（垃圾邮件检测、情感分析、新闻分类）、实时预测。

**优点**：训练和预测速度极快、在文本分类中效果出色、对小数据集友好。  
**缺点**：特征独立假设在现实中很少成立，对特征之间有强关联的数据效果较差。

> 例子：根据邮件中出现的关键词（"中奖"、"免费"、"优惠"等），判断该邮件是否为垃圾邮件。

### K-Means 聚类

**原理**：
1. 随机初始化 K 个簇中心
2. 将每个数据点分配到最近的簇中心
3. 重新计算每个簇的中心（取簇内所有点的均值）
4. 重复步骤 2-3，直到簇中心不再变化（收敛）

**适用场景**：客户分群、图像颜色压缩、数据预分析、异常检测。

**优点**：简单高效、易于实现、适合大规模数据。  
**缺点**：需要预先指定 K 值、对初始中心点敏感、只能发现球形簇、对异常值敏感。

> 例子：电商平台将用户按消费金额和购买频率聚类为"高价值客户"、"潜力客户"和"流失风险客户"三类，以实施不同的营销策略。

## 2.6 学习建议与延伸阅读

### 学习路径建议

1. **打好数学基础**：线性代数（矩阵运算）、概率论与统计、微积分（梯度）——不需要精通，但要理解核心概念
2. **动手实践**：使用 Python + Scikit-learn 从简单项目开始，比如鸢尾花分类、房价预测
3. **循序渐进**：先掌握监督学习的经典算法 → 无监督学习 → 再进入深度学习
4. **多做项目**：Kaggle 竞赛是很好的实战平台，从入门级比赛（Titanic、House Prices）开始
5. **理解原理**：不要仅停留在调用 API 的层面，尝试理解算法背后的数学原理

### 核心要点回顾

```
机器学习 = 数据 + 算法 → 模型
                    ↓
        监督学习：有标签（分类 / 回归）
        无监督学习：无标签（聚类 / 降维）
        强化学习：奖励驱动（策略优化）
                    ↓
        特征工程 + 模型选择 + 评估优化
                    ↓
            部署到实际应用中
```

> **记住**：没有一种算法能解决所有问题（No Free Lunch Theorem）。选择合适的算法取决于你的数据特点、任务需求和资源约束。实践是掌握机器学习的最佳途径！
