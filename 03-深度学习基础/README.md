# 03 - 深度学习基础

## 3.1 从机器学习到深度学习

### 为什么需要深度学习？

传统机器学习（如决策树、SVM）在面对图像、语音、自然语言等**非结构化数据**时，往往力不从心。其核心瓶颈在于——**特征工程**。传统方法需要人类专家手动设计特征（比如识别猫需要先定义"耳朵形状"、"胡须长度"等特征），这既费时又难以覆盖所有情况。

深度学习的革命性突破在于：**让机器自己学习特征**。你只需要把原始数据（如像素值）喂给神经网络，它就能自动从中提取从低级到高级的层次化特征——从边缘、纹理到物体部件，再到完整的物体概念。

### 深度学习的三大驱动力

深度学习的理论基础早在上世纪就已提出，但直到近十几年才真正爆发，这得益于三大驱动力的汇聚：

| 驱动力 | 说明 |
|--------|------|
| **大数据** | 互联网产生了海量标注数据（如 ImageNet 的 1400 万张标注图片），为模型训练提供了"燃料" |
| **大算力** | GPU/TPU 的发展使大规模矩阵运算成为可能，训练时间从数月缩短到数小时 |
| **好算法** | ReLU 激活函数、Batch Normalization、残差连接等技术突破，解决了深层网络难以训练的问题 |

**一句话总结**：深度学习 = 用足够深的神经网络 + 足够多的数据 + 足够强的算力，让机器自动学会"看"、"听"、"读"。

> **2024 补充**：如今预训练数据的规模已远超 ImageNet，如 LAION-5B（50 亿图文对）、The Pile（886GB 文本）等。数据质量和多样性的重要性日益凸显。

## 3.2 神经网络基础

### 感知机——一切的起点

感知机（Perceptron）是最简单的神经网络单元，可以类比为一个**做决策的小裁判**：

1. 接收多个输入信号 $x_1, x_2, ..., x_n$
2. 给每个输入一个**权重** $w_i$（代表重要程度）
3. 加权求和后加上**偏置** $b$
4. 通过**激活函数**决定是否"激活"（输出 1 或 0）

数学表达式：$y = f(w_1x_1 + w_2x_2 + ... + w_nx_n + b)$

单个感知机只能解决**线性可分**的问题。为了处理复杂的非线性问题，我们需要把多个感知机"堆叠"起来。

### 多层感知机 (MLP)

多层感知机就是将感知机组织成多层结构：

```
输入层 → 隐藏层1 → 隐藏层2 → ... → 输出层
```

- **输入层**：接收原始数据
- **隐藏层**：负责特征提取和变换（层数越多，能学到的特征越抽象）
- **输出层**：给出最终预测结果

这就是**"深度"学习中"深度"的来源**——网络层数足够多（深）。

### 激活函数——注入非线性

如果没有激活函数，无论堆叠多少层，网络本质上仍是线性变换。激活函数为网络引入了**非线性能力**，就像给机器装上了"灵活的大脑"。

| 激活函数 | 公式 | 特点 |
|----------|------|------|
| **Sigmoid** | $\sigma(x) = \frac{1}{1+e^{-x}}$ | 输出在 (0,1)，适合概率输出；但存在梯度消失问题 |
| **Tanh** | $\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$ | 输出在 (-1,1)，零中心化；仍有梯度消失问题 |
| **ReLU** | $f(x) = \max(0, x)$ | 计算简单，有效缓解梯度消失；当前最常用的激活函数 |
| **Softmax** | $\frac{e^{x_i}}{\sum_j e^{x_j}}$ | 将输出转化为概率分布，常用于多分类任务的输出层 |

### 前向传播与反向传播

- **前向传播（Forward Propagation）**：数据从输入层出发，逐层经过计算，最终到达输出层得到预测结果。就像水流从源头流向终点。
- **反向传播（Backpropagation）**：计算预测值与真实值之间的误差（损失），然后将误差信号**从输出层反向传递**回每一层，利用链式法则计算每个参数的梯度，从而指导参数更新。就像考试后"倒推"哪里出了错。

这两个过程不断交替进行，就是神经网络的训练过程。

### 损失函数——衡量"错得有多离谱"

损失函数量化了预测值与真实值之间的差距：

- **均方误差 (MSE)**：$L = \frac{1}{n}\sum(y_{pred} - y_{true})^2$，常用于回归任务
- **交叉熵 (Cross-Entropy)**：$L = -\sum y_{true} \log(y_{pred})$，常用于分类任务

### 优化器——如何更新参数

优化器决定了参数沿着梯度方向"走多远、怎么走"：

- **SGD（随机梯度下降）**：最基础的方法，每次用一小批数据计算梯度并更新。简单但可能震荡。
- **Adam**：结合了动量（Momentum）和自适应学习率，是目前最流行的优化器。就像一个"聪明的登山者"，不仅记住了之前走的方向，还能根据地形自动调整步伐大小。

## 3.3 卷积神经网络 (CNN)

### 核心思想

如果用全连接网络处理一张 224×224 的彩色图片，仅第一层就需要 224×224×3 ≈ 15 万个连接，参数量巨大且容易过拟合。CNN 引入了两个关键思想来解决这个问题：

- **局部感受野**：每个神经元只关注图像的一小块区域（就像用放大镜逐区域扫描），而非整张图片
- **权值共享**：同一个卷积核在图像的不同位置使用相同的参数，大幅减少参数量

### 核心组件

| 组件 | 作用 | 比喻 |
|------|------|------|
| **卷积层 (Conv)** | 用卷积核（滤波器）扫描图像，提取局部特征 | 用不同"滤镜"去检测边缘、纹理、颜色等 |
| **池化层 (Pooling)** | 对特征图进行下采样，减小尺寸 | "压缩"图像，保留重要信息，丢弃细节 |
| **全连接层 (FC)** | 将提取到的特征汇总，进行最终分类或回归 | 把所有线索综合起来做最终判断 |

### 经典架构演进

```
LeNet (1998)      →  5 层，手写数字识别的开山之作
AlexNet (2012)    →  8 层，首次在 ImageNet 上大幅领先传统方法，深度学习复兴的标志
VGG (2014)        →  16-19 层，证明了"更深的网络 = 更好的效果"
ResNet (2015)     →  152 层，引入残差连接（Skip Connection），解决了深层网络退化问题
```

ResNet 的**残差连接**是一个里程碑式的创新：它允许信息"跳过"中间层直接传递，就像高速公路上的"立交桥"，让梯度可以畅通无阻地流回浅层，从而使训练上百层的网络成为可能。

### 主要应用

- **图像分类**：识别图片中的物体类别
- **目标检测**：定位并识别图片中的多个物体（YOLO、Faster R-CNN）
- **语义分割**：对图片的每个像素进行分类
- **人脸识别**、医学影像分析等

## 3.4 循环神经网络 (RNN)

### 核心思想

CNN 擅长处理空间结构（图像），但对于**序列数据**（文本、语音、时间序列）则需要另一种架构。RNN 的核心思想是：**拥有"记忆"能力**。

标准的前馈网络处理每个输入时是独立的。而 RNN 在处理序列中的每个元素时，会将**上一步的隐藏状态**传递给下一步，就像人在阅读一篇文章时，会记住之前读过的内容来理解当前的句子。

```
h_t = f(W_h · h_{t-1} + W_x · x_t + b)
```

其中 $h_t$ 是当前时刻的隐藏状态，$h_{t-1}$ 是上一时刻的"记忆"。

### 标准 RNN 的致命问题：梯度消失

当序列很长时（比如一段几百字的文章），标准 RNN 的梯度在反向传播过程中会被反复乘以小于 1 的数，导致**梯度指数级衰减**（梯度消失）。结果就是网络"记不住"很久以前的信息——就像一个健忘的人，只记得最近发生的事。

### LSTM 和 GRU——给 RNN 装上"笔记本"

**LSTM（长短期记忆网络）** 通过引入精巧的**门控机制**来解决梯度消失问题：

- **遗忘门**：决定丢弃哪些旧信息（"这条信息不重要了，可以忘掉"）
- **输入门**：决定存储哪些新信息（"这条新信息很重要，记下来"）
- **输出门**：决定输出哪些信息（"根据当前需要，取出相关的记忆"）

**GRU（门控循环单元）** 是 LSTM 的简化版本，将遗忘门和输入门合并为**更新门**，参数更少，训练更快，效果相当。

### 主要应用

- **自然语言处理**：机器翻译、文本生成、情感分析
- **语音识别**：将语音信号转为文字
- **时间序列预测**：股价预测、天气预报

## 3.5 Transformer 架构

### 为什么需要 Transformer？

尽管 LSTM/GRU 缓解了梯度消失问题，但 RNN 家族有一个根本性的局限：**必须逐步处理序列**，无法并行计算。这就像你必须一个字一个字地读完一本书，不能同时看多个段落。

2017 年 Google 发表了划时代的论文《Attention Is All You Need》，提出了 **Transformer** 架构，彻底抛弃了循环结构，完全基于**注意力机制**来处理序列。

### 注意力机制 (Attention)——"重点关注"

注意力机制的核心思想非常直观：**不是所有信息都同等重要，模型应该学会"关注"最相关的部分**。

打个比方：你在嘈杂的聚会中与朋友交谈，你的大脑会自动"聚焦"朋友的声音，而"忽略"背景噪音。这就是注意力机制。

### 自注意力 (Self-Attention)

自注意力是 Transformer 的核心引擎。对于输入序列中的每个元素，它会计算该元素与**序列中所有其他元素**的关联程度。具体步骤：

1. 将每个输入映射为三个向量：**Query (Q)**、**Key (K)**、**Value (V)**
2. 用 Q 和所有 K 做点积运算，得到注意力分数（"谁和我最相关？"）
3. 对分数进行 Softmax 归一化，得到注意力权重
4. 用权重对所有 V 加权求和，得到输出

$$\text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

其中 $\sqrt{d_k}$ 是缩放因子，防止点积值过大。

### Multi-Head Attention——多角度审视

与其只用一组 Q、K、V 计算注意力，不如用**多组**（多个"头"），让模型从不同的**表示子空间**中学习关联关系。就像同时用多个角度观察一个物体，能获得更全面的理解。

### 编码器-解码器结构

Transformer 由**编码器（Encoder）** 和**解码器（Decoder）** 组成：

- **编码器**：理解输入序列，将其编码为语义丰富的表示。由多层"自注意力 + 前馈网络"堆叠而成。
- **解码器**：基于编码器的输出，逐步生成目标序列。除了自注意力外，还有**交叉注意力**层来关注编码器的输出。

### 位置编码 (Positional Encoding)

由于 Transformer 没有循环结构，它天然**不知道序列的先后顺序**。因此需要人为注入位置信息——通过将正弦/余弦函数生成的**位置编码向量**加到输入嵌入上，让模型感知"谁在前、谁在后"。

### 为什么 Transformer 取代了 RNN？

| 对比维度 | RNN | Transformer |
|----------|-----|-------------|
| 并行能力 | 必须逐步计算，无法并行 | 所有位置可同时计算，天然并行 |
| 长距离依赖 | 信息传递需要多步，容易丢失 | 任意两个位置直接关联，一步到位 |
| 训练效率 | 序列越长训练越慢 | 可充分利用 GPU 并行，效率极高 |

**新兴架构：状态空间模型（SSM）**

2024 年，以 **Mamba** 为代表的状态空间模型（State Space Model）引起广泛关注。SSM 具有**线性时间复杂度**（相比 Transformer 的二次复杂度），在长序列处理上更高效。虽然 Transformer 仍是主流架构，但 Mamba 等 SSM 在部分任务上已展现出接近甚至超越 Transformer 的性能，代表了一个值得关注的新方向。

Transformer 已成为当今 AI 的基石架构：**BERT**（编码器）、**GPT**（解码器）、**T5**（编码器-解码器）都是基于 Transformer 的变体。

## 3.6 深度学习的训练技巧

训练一个好的深度学习模型，不仅需要好的架构设计，还需要一系列实用的训练技巧：

### Batch Normalization（批归一化）

对每一层的输入进行归一化处理（使其均值为 0、方差为 1），可以加速训练收敛、减少对参数初始化的敏感性。就像给每一层的数据做了一次"标准化体检"，保证数据分布不会"跑偏"。

### Dropout（随机失活）

训练时随机"关闭"一部分神经元（通常 20%-50%），强迫网络不过度依赖任何单个神经元。这是一种高效的**正则化**方法，能有效防止过拟合。就像团队训练时随机抽掉几个成员，迫使其他人也学会干活。

### 学习率调度

学习率是训练中最重要的超参数之一。常见策略：

- **阶梯衰减**：每隔若干轮将学习率乘以衰减因子
- **余弦退火**：学习率按余弦曲线平滑下降
- **Warmup**：训练初期先用较小的学习率"热身"，再逐渐增大。这对 Transformer 尤其重要

### 数据增强 (Data Augmentation)

通过对训练数据进行随机变换（翻转、旋转、裁剪、颜色抖动等）来**人为扩大训练集**。模型看到更多样化的数据，泛化能力更强。这就像考试前做各种变式题，而不是只背标准答案。

### 迁移学习 (Transfer Learning)

不从零开始训练模型，而是利用在大规模数据集上**预训练**好的模型（如 ImageNet 上训练的 ResNet），在自己的小数据集上进行**微调（Fine-tuning）**。这是当前最实用的技巧之一——站在巨人的肩膀上，用少量数据也能获得优秀的效果。

**参数高效微调（PEFT）**

在大模型时代，全量微调的成本极高（GPT-3 有 1750 亿参数）。2023-2024 年涌现出多种参数高效微调方法：

- **LoRA（Low-Rank Adaptation）**：在模型的权重矩阵旁插入低秩分解矩阵，只训练新增的少量参数（通常不到原模型的 1%），效果接近全量微调
- **QLoRA**：在 LoRA 的基础上结合 4-bit 量化，使得在消费级 GPU（如 24GB 显存的 RTX 4090）上微调 70B 参数的大模型成为可能
- **Adapter**：在 Transformer 层之间插入小型适配器模块，冻结原始参数只训练适配器

这些方法极大降低了大模型微调的门槛，使个人开发者也能定制自己的 AI 模型。

### 早停 (Early Stopping)

在训练过程中，监控验证集上的表现。当验证集损失不再下降（甚至开始上升）时，提前停止训练，防止过拟合。就像煮饭时到了合适的时间就关火，不要煮过头。

## 3.7 学习建议与延伸阅读

### 学习路线建议

1. **打好数学基础**：线性代数（矩阵运算）、概率统计、微积分（偏导数和链式法则）
2. **掌握一个框架**：推荐 **PyTorch**（灵活直观）或 **TensorFlow**（生态完善）
3. **从经典入手**：先跑通 MNIST 手写数字识别（深度学习的 "Hello World"）
4. **动手做项目**：在 Kaggle 上参加竞赛，在实践中深化理解
5. **阅读论文**：从经典论文（AlexNet、ResNet、Attention Is All You Need）开始

### 推荐学习资源

| 类型 | 推荐资源 |
|------|----------|
| 在线课程 | 吴恩达《深度学习专项课程》(Coursera)、李宏毅《机器学习》(YouTube/B站) |
| 经典教材 | 《深度学习》(花书, Goodfellow 等)、《动手学深度学习》(李沐 等) |

### 必读经典论文

- **BERT** (2018)：预训练语言模型，NLP 的范式转换
